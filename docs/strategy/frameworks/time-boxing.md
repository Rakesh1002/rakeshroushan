# Time-Boxing Framework

## Overview

Time-boxing is the key to launching 100 products in 100 days. Every activity has a maximum time limit. When time's up, ship or cut scope.

---

## Daily Time Blocks

### The Ideal Day (12 hours)

```
06:00 - 06:30  Morning routine, metrics check
06:30 - 10:30  Deep Work Block 1 (4 hours, no interruptions)
10:30 - 11:00  Break
11:00 - 12:00  Marketing/Outreach (1 hour)

14:00 - 18:00  Deep Work Block 2 (4 hours)

19:00 - 20:00  Buffer / Overflow
```

## Parallel Product Development

### Running 2 Products in Parallel

```
WEEK STRUCTURE (2 Products)

Monday:    Product A - Core feature
Tuesday:   Product A - Complete MVP
Wednesday: Product A - Launch + Product B - Start
Thursday:  Product A - Support + Product B - Build
Friday:    Product B - Complete MVP
Saturday:  Product B - Polish
Sunday:    Product B - Launch + Weekly review

Result: 2 products launched per week
```

### Time Split

|-----|-----------|-----------|
| Mon | 100% | 0% |
| Tue | 80% | 20% |
| Wed | 50% | 50% |
| Thu | 30% | 70% |
| Fri | 20% | 80% |
| Sat | 10% | 90% |
| Sun | 20% | 80% |

## Scope Cutting Rules

### When Time Runs Out

If you hit the time limit and aren't done:

1. **Stop building the feature**
2. **Cut scope immediately**
3. **Ship what works**
4. **Add to backlog for later**

### Scope Cutting Hierarchy

What to cut first (in order):

```
1. Nice-to-have features
2. Edge case handling
3. Responsive design details
4. Animation/polish
5. Settings/preferences
6. Admin dashboard
7. Multiple auth providers
8. Advanced features

What to NEVER cut:
× Core value feature
× Payment flow
× Basic error handling
× Security essentials
```

### The 80/20 Launch Rule

```
Launch when you have:
✓ 80% of planned functionality
✓ 100% of core value
✓ 0 critical bugs
✓ Basic analytics
✓ Working payment

Don't wait for:
× Perfect UI
× All edge cases
× Full test coverage
× Documentation
× Multiple tiers
```

## Interruption Management

### Interrupt Categories

| Type | Response | Time |
|------|----------|------|
| Emergency (site down) | Handle now | Immediate |
| Customer issue | Batch + respond | 2x/day |
| General email | Batch | 1x/day |
| Social media | Batch | 1x/day |
| Meetings | Avoid or time-box | 30 min max |

### Protected Time

```
RULE: Deep work blocks are sacred.

During 06:30-10:30 and 14:00-18:00:
× No email
× No Slack
× No Twitter
× No meetings
× Phone on DND

Only interruptions allowed:
✓ Production is down
✓ Payment system broken
✓ Security incident
```

### Batching Strategy

```
Email: Check at 11:00 and 19:00 only
Social: Check at 11:00 only
Support: Check at 13:00 only
Messages: Check at 13:00 and 19:00
```

## The Time-Box Mindset

### Core Principles

```
1. Done is better than perfect
2. Constraints breed creativity
3. Shipping is learning
4. Time is fixed, scope is variable
5. Say no to protect your time
```

### Daily Mantras

```
"If it won't get done in the time box, it won't ship."

"The answer to 'how long will this take' is 'how long do I have?'"

"Ship today. Polish tomorrow. Maybe."

"Every hour spent on one thing is an hour not spent on another."
```
